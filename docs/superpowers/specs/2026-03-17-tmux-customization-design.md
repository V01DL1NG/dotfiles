# tmux Customisation TUI ‚Äî Design Spec

**Date:** 2026-03-17
**Status:** Draft
**Scope:** Extend `tmux-config.sh` with a two-stage interactive TUI (fzf toggles ‚Üí select variant menus) that generates `~/.tmux.conf.local`; move `tmux-config.sh` to run last in `install-all.sh`

---

## Overview

`tmux-config.sh` gains an optional customisation step after the base install. The user is asked "Customise tmux? [Y/n]". If yes, a two-stage TUI collects their preferences and writes `~/.tmux.conf.local`. The base `tmux.conf` (repo-managed, symlinked) sources this file on every tmux start, so overrides apply automatically. Re-running `tmux-config.sh` regenerates the local file (timestamped backup first). Deleting it restores pure base config.

---

## Component 1 ‚Äî `tmux.conf` base change

Add one line near the bottom of `tmux.conf`, **before** the TPM bootstrap block:

```tmux
# Source personal overrides if present (generated by tmux-config.sh)
if "test -f ~/.tmux.conf.local" "source-file ~/.tmux.conf.local"
```

This uses tmux's built-in conditional syntax (compatible with tmux 2.6+). The line is placed after all base settings so the local file can override anything above it, but before TPM so plugin declarations in the local file (future use) would still be picked up by TPM.

## Component 2 ‚Äî `tmux-config.sh` additions

### 2a. Entry point

After the existing base install block (tmux install, nowplaying-cli, symlink tmux.conf, symlink now-playing.sh), add:

```bash
echo ""
if [ -t 0 ]; then
  # Prompt only when stdin is a terminal (not when piped)
  printf "  Customise tmux settings? [Y/n] "
  read -r reply
  case "$reply" in
    [Nn]*) info "Skipping customisation ‚Äî base config applied." ;;
    *)     tmux_customise ;;
  esac
else
  info "Non-interactive ‚Äî skipping tmux customisation."
fi
```

### 2b. `tmux_customise` function

Orchestrates both stages and writes the output file:

```
tmux_customise()
  ‚îú‚îÄ‚îÄ Stage 1: fzf_toggle_stage()   ‚Üí populates ENABLED[] and DISABLED[]
  ‚îú‚îÄ‚îÄ Stage 2: variant_stage()      ‚Üí populates VARIANTS{} associative array
  ‚îî‚îÄ‚îÄ write_local_conf()            ‚Üí writes ~/.tmux.conf.local
```

### 2c. Stage 1 ‚Äî fzf toggle checklist

All items pre-selected (current defaults). Uses the same fzf mechanism as `macos-defaults.sh` (`start:select-all` bind). If fzf is not installed, falls back to a `select` preset menu (Full / Minimal / Skip) ‚Äî same fallback pattern as `macos-defaults.sh`.

**Toggle items** (fzf label ‚Üí internal key):

| fzf label | Key | Default |
|-----------|-----|---------|
| `[Mouse]       Enable mouse support` | `mouse` | on |
| `[Mouse]       Smart scroll (pass-through for vim/less)` | `smart_scroll` | on |
| `[Navigation]  Vim-style pane nav (h/j/k/l)` | `vim_nav` | on |
| `[Copy]        Vi copy mode keys` | `vi_copy` | on |
| `[Copy]        Clipboard integration` | `clipboard` | on |
| `[Status Bar]  Now-playing widget` | `nowplaying` | on (macOS only ‚Äî hidden on Linux) |
| `[Status Bar]  Clock` | `clock` | on |
| `[Pane]        Border labels (index + current command)` | `border_labels` | on |
| `[Session]     Session persistence (resurrect + continuum)` | `persistence` | on |

`nowplaying` is only shown on macOS (`$DOTFILES_OS = "macos"`). On Linux it is silently omitted from the list and treated as disabled.

### 2d. Stage 2 ‚Äî variant select menus

Variant menus appear sequentially after fzf closes. Each menu uses `select` (no fzf dependency). Only menus relevant to enabled features are shown ‚Äî disabled features are skipped entirely.

| Menu | Shown when | Choices | Default |
|------|-----------|---------|---------|
| **Prefix key** | always | `Ctrl+a` ¬∑ `Ctrl+b` | `Ctrl+a` |
| **Pane border style** | always | Double ¬∑ Single ¬∑ Rounded | Double |
| **Status bar style** | always | Themed velvet ¬∑ Minimal | Themed velvet |
| **Scrollback buffer** | always | 10,000 ¬∑ 50,000 ¬∑ Unlimited | 10,000 |
| **Copy command** | `clipboard` on | Auto-detect ¬∑ `pbcopy` ¬∑ `xclip` ¬∑ `xsel` | Auto-detect |
| **Now-playing position** | `nowplaying` on | Right ¬∑ Left | Right |
| **Clock position** | `clock` on | Right ¬∑ Left | Right |

**Auto-detect copy command logic:**
- macOS ‚Üí `pbcopy`
- Linux: check `xclip` first, then `xsel`; if neither found, emit a warning and skip clipboard binding

**Minimal status bar style:** Plain segments, no powerline glyphs (`\ue0b4` / `\ue0b6`), no velvet colours ‚Äî suitable for terminals without Nerd Font.

### 2e. `write_local_conf` ‚Äî output file format

Location: `~/.tmux.conf.local`

**Backup:** if an existing `~/.tmux.conf.local` is present, back it up to `~/.tmux.conf.local.backup.YYYYMMDDHHMMSS` before overwriting.

**Content rules:**
- Always write a header comment with generation timestamp and re-run instructions
- Write only settings that differ from base `tmux.conf` defaults **or** are disabled (explicit `off`/`unbind` lines needed to override base)
- Each block has a comment label (e.g. `# ‚îÄ‚îÄ Prefix ‚îÄ‚îÄ‚îÄ`)
- If all choices match base defaults and all features are enabled, write the header comment only ‚Äî no `set` lines

**Sections written (in order):**

1. **Header**
```bash
# ~/.tmux.conf.local ‚Äî generated by tmux-config.sh
# Generated: YYYY-MM-DD HH:MM:SS
# Re-run ./tmux-config.sh to regenerate (backs up this file first).
# Delete this file to restore base config defaults.
```

2. **Prefix** ‚Äî only written if not `Ctrl+a`:
```tmux
unbind C-a
set -g prefix C-b
bind C-b send-prefix
```

3. **Mouse** ‚Äî only written if disabled:
```tmux
set -g mouse off
```

4. **Smart scroll** ‚Äî only written if disabled (removes the WheelUpPane/WheelDownPane binds):
```tmux
unbind -n WheelUpPane
unbind -n WheelDownPane
```

5. **Vim nav** ‚Äî only written if disabled:
```tmux
unbind h
unbind j
unbind k
unbind l
```

6. **Vi copy mode** ‚Äî only written if disabled:
```tmux
setw -g mode-keys emacs
```

7. **Clipboard** ‚Äî written if enabled (with resolved copy command) or disabled:
```tmux
# enabled:
bind -T copy-mode-vi y send -X copy-pipe-and-cancel "pbcopy"
bind -T copy-mode-vi MouseDragEnd1Pane send -X copy-pipe-and-cancel "pbcopy"
# disabled:
unbind -T copy-mode-vi y
unbind -T copy-mode-vi MouseDragEnd1Pane
```

8. **Scrollback** ‚Äî only written if not 10,000:
```tmux
set -g history-limit 50000   # or 0 for unlimited
```

9. **Pane border style** ‚Äî only written if not Double:
```tmux
set -g pane-border-lines single   # or rounded
```

10. **Border labels** ‚Äî only written if disabled:
```tmux
set -g pane-border-status off
```

11. **Status bar** ‚Äî written when style is Minimal, or when now-playing/clock positions differ from base:

*Minimal style* overrides (replaces all velvet powerline segments):
```tmux
set -g status-left " #S "
set -g status-left-length 20
set -g status-right " %H:%M "
set -g status-right-length 15
```

*Themed style with positional changes* ‚Äî rewrite only the affected `status-left` / `status-right` / `status-left-length` / `status-right-length` lines. The four possible combinations (now-playing left/right √ó clock left/right) each have a defined template. If now-playing is disabled, its widget is omitted from both sides.

12. **Session persistence** ‚Äî only written if disabled:
```tmux
set -g @plugin 'tmux-plugins/tmux-resurrect'   # removed
set -g @plugin 'tmux-plugins/tmux-continuum'   # removed
```

Wait ‚Äî TPM plugins cannot be "unset" via `source-file`. Instead, when persistence is disabled, write a note comment and rely on `tmux-config.sh` to skip installing those plugins during the base setup. See Component 3.

**Revised persistence handling:** `persistence` toggle affects `tmux-config.sh` directly (controls whether resurrect/continuum lines are written to `tmux.conf` on install), not `~/.tmux.conf.local`. See Component 3.

## Component 3 ‚Äî Plugin management for persistence toggle

TPM plugins are declared in `tmux.conf` (the repo file). Since `tmux.conf` is symlinked and cannot be personalised per-user, persistence cannot be toggled via `~/.tmux.conf.local`.

**Solution:** `tmux-config.sh` writes a minimal `~/.tmux.conf.plugins` file that is sourced by `tmux.conf` (via an additional `if "test -f ~/.tmux.conf.plugins" "source-file ~/.tmux.conf.plugins"` line in the base, placed just before the TPM bootstrap block). When persistence is enabled, `~/.tmux.conf.plugins` contains the resurrect/continuum plugin declarations. When disabled, the file is written empty (or absent).

This keeps plugin declarations out of the shared `tmux.conf` while allowing per-user plugin selection.

**`tmux.conf` changes (revised):**
```tmux
# Source personal overrides if present
if "test -f ~/.tmux.conf.local" "source-file ~/.tmux.conf.local"

# Source personal plugin declarations if present
if "test -f ~/.tmux.conf.plugins" "source-file ~/.tmux.conf.plugins"

# Bootstrap TPM if not installed
if "test ! -d ~/.tmux/plugins/tpm" \
   "run 'git clone https://github.com/tmux-plugins/tpm ~/.tmux/plugins/tpm && ~/.tmux/plugins/tpm/bin/install_plugins'"

# Initialize TPM (keep at very bottom)
run '~/.tmux/plugins/tpm/tpm'
```

The existing hardcoded resurrect/continuum declarations in `tmux.conf` are **removed** from the base file and moved into `~/.tmux.conf.plugins` generation in `tmux-config.sh`.

## Component 4 ‚Äî `install-all.sh` changes

`tmux-config.sh` is removed from the `scripts[]` loop and added as an explicit call at the end, just before `macos-defaults.sh`:

```bash
# tmux ‚Äî moved to end for interactive customisation prompt
echo ""
echo "-- [Tmux + customisation] --"
if [ -f "$SCRIPT_DIR/tmux-config.sh" ]; then
  bash "$SCRIPT_DIR/tmux-config.sh"
else
  echo "Warning: tmux-config.sh not found, skipping."
fi
```

Order at end of `install-all.sh`:
1. `setup.sh` (oh-my-posh)
2. `iterm-config.sh` (macOS only)
3. `tmux-config.sh` ‚Üê moved here
4. `macos-defaults.sh` (macOS only)
5. "Setup complete!" banner

## Component 5 ‚Äî fzf fallback

When fzf is not installed, Stage 1 degrades to a `select` preset menu:

```
1) Full     ‚Äî all features enabled, current defaults
2) Minimal  ‚Äî mouse on, vim nav on, no status bar bling, no persistence
3) Custom   ‚Äî not available without fzf (message printed)
4) Skip     ‚Äî no customisation
```

Warning printed: `"fzf not found ‚Äî install it for per-feature selection (tools-config.sh installs it automatically)"`

For "Minimal" preset, Stage 2 variant menus still run (prefix key, border style, scrollback, copy command remain user choices even in fallback mode).

## Error handling

| Condition | Behaviour |
|-----------|-----------|
| Non-interactive stdin (piped) | Skip customisation silently with info message |
| fzf not installed | Fall back to select preset menu (Component 5) |
| No features selected in fzf | Write header-only `~/.tmux.conf.local`, print info |
| Auto-detect copy: no clipboard tool found (Linux) | Warn, skip clipboard binding |
| Existing `~/.tmux.conf.local` | Back up with timestamp before overwriting |
| `tmux.conf.local` / `tmux.conf.plugins` absent at tmux start | Base config applies as-is (graceful ‚Äî conditional source) |

## Testing

- `bash -n tmux-config.sh` ‚Äî syntax clean
- Dry-run variant: `TMUX_DRY_RUN=1 bash tmux-config.sh` ‚Äî prints generated file contents to stdout, writes nothing
- Verify `~/.tmux.conf.local` is sourced: `tmux source ~/.tmux.conf && tmux show -g prefix` reflects override
- Verify backup created on re-run
- Verify non-macOS: `nowplaying` absent from fzf list, auto-detect copy uses xclip/xsel
- Verify persistence disabled: `~/.tmux.conf.plugins` empty, resurrect/continuum not loaded
