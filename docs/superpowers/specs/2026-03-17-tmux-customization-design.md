# tmux Customisation TUI — Design Spec

**Date:** 2026-03-17
**Status:** Draft
**Scope:** Extend `tmux-config.sh` with a two-stage interactive TUI (fzf toggles → select variant menus) that generates `~/.tmux.conf.local`; move `tmux-config.sh` to run last in `install-all.sh`

---

## Overview

`tmux-config.sh` gains an optional customisation step after the base install. The user is asked "Customise tmux? [Y/n]". If yes, a two-stage TUI collects their preferences and writes `~/.tmux.conf.local`. The base `tmux.conf` (repo-managed, symlinked) sources this file on every tmux start, so overrides apply automatically. Re-running `tmux-config.sh` regenerates the local file (timestamped backup first). Deleting it restores pure base config.

---

## Component 1 — `tmux.conf` base changes

Two lines are added near the bottom of `tmux.conf`, **before** the TPM bootstrap block:

```tmux
# Source personal overrides if present (generated by tmux-config.sh)
if "test -f ~/.tmux.conf.local" "source-file ~/.tmux.conf.local"

# Source personal plugin declarations if present (generated by tmux-config.sh)
if "test -f ~/.tmux.conf.plugins" "source-file ~/.tmux.conf.plugins"
```

Both use tmux's built-in conditional syntax (compatible with tmux 2.6+, which covers Ubuntu 18.04 LTS and later). Lines are placed after all base settings so the local files can override anything above them, and before TPM so plugin declarations in `~/.tmux.conf.plugins` are picked up by TPM.

The existing hardcoded resurrect/continuum `set -g @plugin` declarations and their `@resurrect-*` / `@continuum-*` option lines are **removed** from `tmux.conf` and moved exclusively to `~/.tmux.conf.plugins` generation (see Component 3).

## Component 2 — `tmux-config.sh` additions

### 2a. Entry point

After the existing base install block (tmux install, nowplaying-cli, symlink tmux.conf, symlink now-playing.sh), add:

```bash
echo ""
if [ -t 0 ]; then
  # Prompt only when stdin is a terminal (not when piped)
  printf "  Customise tmux settings? [Y/n] "
  read -r reply
  case "$reply" in
    [Nn]*) info "Skipping customisation — base config applied." ;;
    *)     tmux_customise ;;
  esac
else
  info "Non-interactive — skipping tmux customisation."
fi
```

When the user skips customisation and `~/.tmux.conf.plugins` does not yet exist, `tmux-config.sh` writes it with persistence **enabled** as the safe default — preserving the same behaviour as the current base config (see Component 3).

`--dry-run` flag: if `tmux-config.sh` is called with `--dry-run`, the TUI runs normally but `write_local_conf()` and `write_plugins_conf()` print the generated file contents to stdout instead of writing to disk. Consistent with `macos-defaults.sh` convention. `install-all.sh` does **not** forward `--dry-run` to `tmux-config.sh`; invoke `tmux-config.sh` directly for dry-run mode.

### 2b. `tmux_customise` function

Orchestrates both stages and writes the output files:

```
tmux_customise()
  ├── Stage 1: fzf_toggle_stage()   → populates ENABLED[] and DISABLED[]
  ├── Stage 2: variant_stage()      → populates VARIANTS{} associative array
  ├── write_local_conf()            → writes ~/.tmux.conf.local
  └── write_plugins_conf()          → writes ~/.tmux.conf.plugins
```

### 2c. Stage 1 — fzf toggle checklist

All items pre-selected (current defaults). Uses the same fzf mechanism as `macos-defaults.sh` (`start:select-all` bind). If fzf is not installed, falls back to a `select` preset menu (Full / Minimal / Skip) — see Component 5.

**Toggle items** (fzf label → internal key):

| fzf label | Key | Default |
|-----------|-----|---------|
| `[Mouse]       Enable mouse support` | `mouse` | on |
| `[Mouse]       Smart scroll (pass-through for vim/less)` | `smart_scroll` | on |
| `[Navigation]  Vim-style pane nav (h/j/k/l)` | `vim_nav` | on |
| `[Copy]        Vi copy mode keys` | `vi_copy` | on |
| `[Copy]        Clipboard integration` | `clipboard` | on |
| `[Status Bar]  Now-playing widget` | `nowplaying` | on (macOS only) |
| `[Status Bar]  Clock` | `clock` | on |
| `[Pane]        Border labels (index + current command)` | `border_labels` | on |
| `[Session]     Session persistence (resurrect + continuum)` | `persistence` | on |

`nowplaying` is only shown on macOS (`$DOTFILES_OS = "macos"`). On Linux it is silently omitted from the list and treated as disabled.

### 2d. Stage 2 — variant select menus

Variant menus appear sequentially after fzf closes. Each menu uses bash `select`. All `select` loops include a Ctrl-D / EOF guard: the loop body uses `|| true` and the PS3 prompt is reset after each use to prevent `set -euo pipefail` from aborting on Ctrl-D input. Only menus relevant to enabled features are shown.

| Menu | Shown when | Choices | Default |
|------|-----------|---------|---------|
| **Prefix key** | always | `Ctrl+a` · `Ctrl+b` | `Ctrl+a` |
| **Pane border style** | always | Double · Single · Heavy | Double |
| **Status bar style** | always | Themed velvet · Minimal | Themed velvet |
| **Scrollback buffer** | always | 10,000 · 50,000 · Unlimited | 10,000 |
| **Copy command** | `clipboard` on | Auto-detect · `pbcopy` · `xclip` · `xsel` | Auto-detect |
| **Now-playing position** | `nowplaying` on | Right · Left | Right |
| **Clock position** | `clock` on | Right · Left | Right |

**Auto-detect copy command logic:**
- macOS → `pbcopy`
- Linux: check `xclip` first, then `xsel`; if neither found, emit a warning and skip clipboard binding

**Minimal status bar style:** Plain segments, no powerline glyphs (`\ue0b4` / `\ue0b6`), no velvet colours — suitable for terminals without Nerd Font.

### 2e. `write_local_conf` — output file format

Location: `~/.tmux.conf.local`

**Backup:** if an existing `~/.tmux.conf.local` is present, back it up to `~/.tmux.conf.local.backup.YYYYMMDDHHMMSS` before overwriting.

**Content rules:**
- Always write a header comment with generation timestamp and re-run instructions
- Write only settings that differ from base `tmux.conf` defaults **or** are disabled (explicit `off`/`unbind` lines needed to override base)
- On Linux, **always** write the clipboard section (either the correct xclip/xsel override, or an unbind if disabled) — the base `tmux.conf` hardcodes `pbcopy` which is macOS-only
- Each block has a comment label (e.g. `# ── Prefix ───`)
- If all choices match base defaults, all features are enabled, and clipboard was not overridden for Linux, write the header comment only — no `set` lines

**Sections written (in order):**

1. **Header**
```tmux
# ~/.tmux.conf.local — generated by tmux-config.sh
# Generated: YYYY-MM-DD HH:MM:SS
# Re-run ./tmux-config.sh to regenerate (backs up this file first).
# Delete this file to restore base config defaults.
```

2. **Prefix** — only written if not `Ctrl+a`:
```tmux
unbind C-a
set -g prefix C-b
bind C-b send-prefix
```

3. **Mouse** — only written if disabled:
```tmux
set -g mouse off
```

4. **Smart scroll** — only written if disabled:
```tmux
unbind -n WheelUpPane
unbind -n WheelDownPane
```

5. **Vim nav** — only written if disabled:
```tmux
unbind h
unbind j
unbind k
unbind l
```

6. **Vi copy mode** — only written if disabled:
```tmux
setw -g mode-keys emacs
```

7. **Clipboard** — written if enabled with resolved copy command, or if disabled, or always on Linux:
```tmux
# enabled (example with xclip):
bind -T copy-mode-vi y send -X copy-pipe-and-cancel "xclip -selection clipboard"
bind -T copy-mode-vi MouseDragEnd1Pane send -X copy-pipe-and-cancel "xclip -selection clipboard"
# disabled:
unbind -T copy-mode-vi y
unbind -T copy-mode-vi MouseDragEnd1Pane
```

8. **Scrollback** — only written if not 10,000:
```tmux
set -g history-limit 50000   # or 0 for unlimited
```

9. **Pane border style** — only written if not Double:
```tmux
set -g pane-border-lines single   # or heavy
```

10. **Border labels** — only written if disabled:
```tmux
set -g pane-border-status off
```

11. **Status bar** — written when style is Minimal, or when now-playing/clock positions differ from base defaults, or when now-playing is disabled (to remove it from the right side).

*Minimal style* overrides (replaces all velvet powerline segments):
```tmux
set -g status-left " #S "
set -g status-left-length 20
set -g status-right " %H:%M "
set -g status-right-length 15
```

*When now-playing is disabled, clock right (macOS only):*
```tmux
set -g status-right "#[fg=#4c1f5e]\ue0b6#[bg=#4c1f5e,fg=#EFDCF9]  %H:%M "
set -g status-right-length 15
```

*When now-playing is disabled, clock left (macOS only):*
```tmux
set -g status-left "#[bg=#69307A,fg=#EFDCF9,bold]  #S #[bg=#170B3B,fg=#69307A]\ue0b4 #[fg=#4c1f5e]\ue0b6#[bg=#4c1f5e,fg=#EFDCF9]  %H:%M "
set -g status-left-length 60
set -g status-right ""
set -g status-right-length 0
```

*When both now-playing and clock are disabled (macOS only):*
```tmux
set -g status-right ""
set -g status-right-length 0
```

*Themed style positional templates* — the exact `status-left`, `status-right`, `status-left-length`, and `status-right-length` values for all four combinations of (now-playing position) × (clock position), when both are enabled. "Base default" (both right) requires no override; the other three are written in full:

**Both right** (base default — no override written):
```
status-left:        "#[bg=#69307A,fg=#EFDCF9,bold]  #S #[bg=#170B3B,fg=#69307A]\ue0b4"
status-left-length: 25
status-right:       "#[fg=#341948]\ue0b6#[bg=#341948,fg=#EFDCF9] #($HOME/.tmux/now-playing.sh) #[fg=#4c1f5e]\ue0b6#[bg=#4c1f5e,fg=#EFDCF9]  %H:%M "
status-right-length: 70
```

**Now-playing right, clock left:**
```tmux
set -g status-left "#[bg=#69307A,fg=#EFDCF9,bold]  #S #[bg=#170B3B,fg=#69307A]\ue0b4 #[fg=#4c1f5e]\ue0b6#[bg=#4c1f5e,fg=#EFDCF9]  %H:%M "
set -g status-left-length 50
set -g status-right "#[fg=#341948]\ue0b6#[bg=#341948,fg=#EFDCF9] #($HOME/.tmux/now-playing.sh) "
set -g status-right-length 60
```

**Now-playing left, clock right:**
```tmux
set -g status-left "#[bg=#69307A,fg=#EFDCF9,bold]  #S #[bg=#170B3B,fg=#69307A]\ue0b4 #[fg=#341948]\ue0b6#[bg=#341948,fg=#EFDCF9] #($HOME/.tmux/now-playing.sh) "
set -g status-left-length 80
set -g status-right "#[fg=#4c1f5e]\ue0b6#[bg=#4c1f5e,fg=#EFDCF9]  %H:%M "
set -g status-right-length 15
```

**Both left:**
```tmux
set -g status-left "#[bg=#69307A,fg=#EFDCF9,bold]  #S #[bg=#170B3B,fg=#69307A]\ue0b4 #[fg=#341948]\ue0b6#[bg=#341948,fg=#EFDCF9] #($HOME/.tmux/now-playing.sh) #[fg=#4c1f5e]\ue0b6#[bg=#4c1f5e,fg=#EFDCF9]  %H:%M "
set -g status-left-length 100
set -g status-right ""
set -g status-right-length 0
```

## Component 3 — Plugin management (`~/.tmux.conf.plugins`)

TPM plugins cannot be toggled per-user via `~/.tmux.conf.local` because `set -g @plugin` declarations sourced after TPM initialises have no effect. The solution is `~/.tmux.conf.plugins`, sourced by `tmux.conf` just before the TPM bootstrap (Component 1).

**`write_plugins_conf()` behaviour:**

- **Persistence enabled:** write resurrect + continuum declarations and their config options:
```tmux
# ~/.tmux.conf.plugins — generated by tmux-config.sh
set -g @plugin 'tmux-plugins/tmux-resurrect'
set -g @plugin 'tmux-plugins/tmux-continuum'
set -g @resurrect-capture-pane-contents 'on'
set -g @continuum-restore 'on'
set -g @continuum-save-interval '15'
```

- **Persistence disabled:** write header comment only (empty file — TPM still loads but resurrect/continuum are not declared):
```tmux
# ~/.tmux.conf.plugins — generated by tmux-config.sh
# Session persistence disabled — re-run ./tmux-config.sh to enable.
```

**Backup:** back up existing `~/.tmux.conf.plugins` to `~/.tmux.conf.plugins.backup.YYYYMMDDHHMMSS` before overwriting.

**Safe default on skip:** when the user skips customisation (`[Nn]` reply or non-interactive) and `~/.tmux.conf.plugins` does not yet exist, `tmux-config.sh` writes it with persistence enabled (no backup needed — file is new). If it already exists, it is left untouched — no backup, no overwrite.

On the full TUI path (user said Y and completed both stages), `write_plugins_conf()` always backs up any existing `~/.tmux.conf.plugins` and overwrites it with the user's choice.

**Note on plugin unloading:** changing persistence state takes effect on the next tmux server start, not on `source-file` reload. `tmux-config.sh` prints a reminder: `"Restart your tmux server for plugin changes to take effect: tmux kill-server && tmux"`. This reminder is only shown when the persistence choice differs from the current state of `~/.tmux.conf.plugins`.

## Component 4 — `install-all.sh` changes

`tmux-config.sh` is **removed** from the `scripts[]` array (delete the `"tmux-config.sh:Tmux + now-playing"` entry) and added as an explicit call at the end, just before `macos-defaults.sh`:

```bash
# tmux — moved to end for interactive customisation prompt
echo ""
echo "-- [Tmux + customisation] --"
if [ -f "$SCRIPT_DIR/tmux-config.sh" ]; then
  bash "$SCRIPT_DIR/tmux-config.sh"
else
  echo "Warning: tmux-config.sh not found, skipping."
fi
```

Order at end of `install-all.sh`:
1. `setup.sh` (oh-my-posh)
2. `iterm-config.sh` (macOS only)
3. `tmux-config.sh` ← moved here (interactive customisation)
4. `macos-defaults.sh` (macOS only, interactive)
5. "Setup complete!" banner

## Component 5 — fzf fallback

When fzf is not installed, Stage 1 degrades to a `select` preset menu:

```
1) Full     — all features enabled, current defaults
2) Minimal  — mouse on, vim nav on, no status bar bling, no persistence
3) Skip     — no customisation
```

Warning printed: `"fzf not found — install it for per-feature selection (tools-config.sh installs it automatically)"`

**Preset key mappings (sets ENABLED[] / DISABLED[] before Stage 2):**

- **Full:** all 9 keys enabled; on Linux `nowplaying` is always excluded (treated as disabled)
- **Minimal:** enabled = `{mouse, smart_scroll, vim_nav, vi_copy, clipboard, clock}`; disabled = `{nowplaying, border_labels, persistence}`; status bar style variant pre-set to Minimal

For both presets, Stage 2 variant menus still run (prefix key, border style, scrollback, copy command, and any position menus for enabled features are user choices even in fallback mode).

## Error handling

| Condition | Behaviour |
|-----------|-----------|
| Non-interactive stdin (piped) | Skip customisation; write `~/.tmux.conf.plugins` with persistence enabled if absent |
| fzf not installed | Fall back to select preset menu (Component 5) |
| No features selected in fzf | Write header-only `~/.tmux.conf.local`, write plugins conf, print info |
| Auto-detect copy: no clipboard tool found (Linux) | Warn, skip clipboard binding lines |
| Existing `~/.tmux.conf.local` | Back up with timestamp before overwriting |
| Existing `~/.tmux.conf.plugins` | Back up with timestamp before overwriting |
| Local files absent at tmux start | Base config applies as-is (graceful — conditional source) |
| Persistence state change | Print restart reminder (`tmux kill-server && tmux`) |
| `--dry-run` flag | TUI runs normally; `write_local_conf` and `write_plugins_conf` print to stdout only |

## Testing

- `bash -n tmux-config.sh` — syntax clean
- `bash tmux-config.sh --dry-run` — prints generated file contents to stdout, writes nothing
- Verify `~/.tmux.conf.local` is sourced: `tmux source ~/.tmux.conf && tmux show -g prefix` reflects override
- Verify backup created on re-run for both `~/.tmux.conf.local` and `~/.tmux.conf.plugins`
- Verify non-macOS: `nowplaying` absent from fzf list, clipboard section always written with xclip/xsel
- Verify persistence disabled: `~/.tmux.conf.plugins` header-only, resurrect/continuum not loaded by TPM
- Verify skip path: `~/.tmux.conf.plugins` written with persistence enabled when absent
- Verify all four themed status-bar templates produce correct output in dry-run
