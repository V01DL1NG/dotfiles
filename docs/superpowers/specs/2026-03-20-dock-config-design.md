# Dock Configuration TUI — Design Spec

**Date:** 2026-03-20
**Status:** Approved

---

## Problem

On a new Mac the Dock contains Apple defaults. Every time a new machine is set up,
apps must be manually pinned and reordered. There is no way to reproduce a known
Dock layout automatically. The existing `role.sh` system switches shell environment
but leaves the Dock unchanged.

---

## Goals

- Interactive TUI to choose which apps appear in the Dock, per role (work/personal)
- Config persists in the repo as plain text files — editable by hand, re-applicable non-interactively
- `role.sh apply work` automatically applies the saved work Dock layout
- Follows the `tmux-config.sh` TUI pattern (fzf multi-select + `select` variant choices)
- `--dry-run` support throughout
- Idempotent: safe to re-run

---

## Non-Goals

- Linux support (macOS only — Dock is macOS-specific)
- Per-machine overrides (one config per role, same on every Mac)
- Automatic app installation (only apps already in `/Applications` or `~/Applications` are offered)
- Dock icon ordering via drag (dockutil handles order by insertion sequence)

---

## Config Files

Two plain-text files committed to the repo:

```
dock/work.txt
dock/personal.txt
```

Format — one entry per line:
- App path: `/Applications/Arc.app`
- Spacer: `---`
- Comment (ignored): `# work tools`
- Blank lines ignored

Example `dock/work.txt`:
```
/Applications/Arc.app
/Applications/Cursor.app
/Applications/iTerm.app
---
/Applications/Slack.app
/Applications/Zoom.app
---
/Applications/Finder.app
```

Files are generated by the TUI and committed to the repo. They can also be edited
by hand without running the TUI.

## Script: `dock-config.sh`

### Modes

```
./dock-config.sh              # interactive TUI
./dock-config.sh --apply work     # silently apply saved work config
./dock-config.sh --apply personal # silently apply saved personal config
./dock-config.sh --dry-run        # show dockutil commands, no changes
```

### Interactive TUI Flow

**Entry point** — Y/n prompt (same pattern as tmux-config.sh):
```
Customise Dock layout? [Y/n]
```

**Stage 1 — Role selection** — `select` menu:
```
Which role to configure?
  1) Work
  2) Personal
  3) Both
```
Selecting "Both" runs the app picker twice in sequence.

**Stage 2 — App picker** — fzf multi-select (fallback: numbered list if fzf not found):
- Scans `/Applications` and `~/Applications` for installed `.app` bundles
- Presents sorted list; user searches and space-selects desired apps
- No pre-selection — starts empty (unlike tmux-config.sh which pre-selects all)
- Prompt: `Select apps for <Role> Dock (space to select, Enter to confirm)`

**Stage 3 — Variant choices** — `select` loops:
1. Dock position: `Bottom` (default) / `Left` / `Right`
2. Spacers between groups: `Yes` / `No`
3. Clear existing Dock before applying: `Yes` (default) / `No`

If spacers are enabled, the script inserts a `---` spacer line between each group
of consecutively-selected apps (grouping by the order they appear in the fzf output).

**Saves config** to `dock/<role>.txt` then immediately applies.

### `--apply <role>` Flow (non-interactive)

1. Check `dock/<role>.txt` exists — if not, `warn` and exit 0 (never fails role switch)
2. Install `dockutil` via Homebrew if not present
3. Run `dockutil --remove all --no-restart` to clear current Dock
4. For each line in config: skip comments/blanks, add spacer or app via `dockutil`
5. `killall Dock` to apply changes

### `--dry-run`

Prints the `dockutil` commands that would run without executing them. Works for
both interactive and `--apply` modes.

### Dependency: `dockutil`

Added to `Brewfile`. Script checks for `dockutil` and installs via `brew install dockutil`
if missing before any Dock operations.

---

## `role.sh` Integration

In `role.sh apply <role>`, after the zshrc update, add:

```bash
if [ -f "$SCRIPT_DIR/dock-config.sh" ]; then
  bash "$SCRIPT_DIR/dock-config.sh" --apply "$role"
fi
```

If `dock/<role>.txt` does not exist, `dock-config.sh --apply` warns and exits 0 —
the role switch completes successfully regardless.

## Testing: `test-dock-config.sh`

Follows the `test-macos-defaults.sh` / `test-touchid-sudo.sh` pattern.
macOS-only (exits 0 with skip message on Linux).

**Test sections:**

1. **Syntax** — `bash -n dock-config.sh`

2. **`--dry-run` output** — verify:
   - dockutil commands appear in output
   - `--remove all` appears (clear step)
   - App paths from a fixture config appear in output

3. **Arg parsing**:
   - `--apply` with missing config file exits 0 (warn, not error)
   - Invalid args exit non-zero
   - `--dry-run` exits 0

## Risks and Mitigations

| Risk | Mitigation |
|---|---|
| `dockutil` not installed | Auto-install via brew before any Dock operations |
| `dock/<role>.txt` missing at role apply time | Exit 0 with warning — role switch never blocked |
| App path in config no longer exists | Skip missing paths with a warning, continue applying rest |
| Dock cleared but apply fails mid-way | `--dry-run` for preview; user can re-run to restore |
| fzf not available | Fallback to numbered `select` menu (same as tmux-config.sh) |
